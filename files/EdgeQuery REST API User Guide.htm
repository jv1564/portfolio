The Customer Data API (CDAPI) is a REST API with custom reporting endpoints implemented for customers. This documentation describes version 1 (/v1) of the API. New releases may increase the version number.

API Version

Versioning is reflected in the URL of the application.

Example

https://llnw.api.com/v1/c/sw312s/reports/usage

  • The v1 prefix is a version specifier that must be present in url. When significant changes are made to the API (changes that would break compatibility with existing applications) this value will change.

  • The API version is independent of the release number.

Security

Authentication

The REST APIs Authentication approach is used be consistent with other public REST APIs.

Authorization

Access is allowed to customer users with permission to endpoints configured for that user's company.

Rate Limiting

The Rate Limiting policy limits the number of requests certain Report endpoint accepts within a period of time. The response headers tell you everything you need to know about the current rate limit status of a requested report endpoint:

Header Name

Header Name

X-RateLimit-Limit

The maximum number of requests that the report endpoint can serve in current window.

X-RateLimit-Remaining

The number of requests remaining in the current rate limit window.

X-RateLimit-Reset

A Unix timestamp detailing when the rate limit will reset

If you request data from report endpoint that went over the rate limit, you will receive an error response:

Example

HTTP/1.1 429
X-Request-ID: ac1cbac1-0bc1-4635-ba15-64ea884a3bfd
X-RateLimit-Limit: 3
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1488885287
Content-Type: application/json;charset=UTF-8
Transfer-Encoding: chunked
Date: Tue, 07 Mar 2017 11:14:44 GMT
{"timestamp":1488885284653,"request_id":"ac1cbac1-0bc1-4635-ba15-64ea884a3bfd","error":"Too many requests","description":"API rate limit exceeded."}

All endpoints require authentication, so there is no concept of unauthenticated calls and rate limits.

Pagination

CDAPI Pagination is used in the Report endpoint, and is based on the limit and offset request parameters. Limit is the maximum number of rows that may be returned in response. Offset specifies how many records should be skipped.

Example

Request

GET /v1/c/ac1cbac1/reports/usage/storage?limit=30&offset=100 HTTP/1.1

Response

returns 30 or less rows, starting from 101 row. (101, 102, 103 ... 130)

Every Report response has a custom limit response size response_rows_limit (maximum number of rows per one request). When you make a call to Report, API compares the parameter limit in the URL with the response_rows_limit.  If the URL limit value is greater than response_rows_limit, the CDAPI responds with 400 Bad Request.

By default, the CDAPI uses limit which is equal to response_rows_limit and offset=0.
